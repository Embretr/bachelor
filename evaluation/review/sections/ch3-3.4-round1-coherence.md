## Critical issues

### CRITICAL 1 — "themes" used for two structurally different objects within ¶1

¶1 sentence-1: *"...across the five themes that anchored each call (current tools and workflow, assignment criteria, sick-leave handling, attitudes to automation, and adoption criteria)"* — interview-guide topics, pre-coded scaffolds (named in §3.3 ¶3 as "the same five themes anchored every conversation").

¶1 later: *"The analysis identified five themes across the seven transcripts; following \textcite[Table~2, criterion~15]{braun2006thematic}, those themes were named through the coding process rather than treated as having surfaced unaided"* — analytic outputs of the six-phase TA process.

Same word "themes", same count (five), used for two structurally different object types. This silently equates pre-coded scaffolds with constructed analytic outputs, contradicting the inductive positioning the same paragraph claims (codes drawn "without a pre-existing coding frame"). Either the analysis was inductive and the convergence on the same five labels is suspicious, or the analysis lifted the guide topics directly and is in fact deductive in semantic-mapping practice.

**Fix:** Rename the two object types so the reader can distinguish them. Option A — name the analytic-output themes explicitly ("the analysis identified themes including the resource-utilization visibility gap, the operator-vs-owner asymmetry, automation scepticism with override preference"). Option B — keep the count and acknowledge convergence-with-extension ("the analysis produced themes elaborating each of the five guide topics with cross-cutting findings such as the visibility gap and the operator-vs-owner asymmetry"). Source notes for `braun2006thematic` provide canonical analytic-theme names usable in Option A.
