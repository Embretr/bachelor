## Suggestions (1 lessons-learned candidate)

### Em-dash budget for narrative method/findings sections
**Proposed rule:** Sections that use a six-bullet iteration / finding / decision structure (currently §3.5, anticipated §4.5 / §4.7 / §5.1.x / §5.4 L#) should cap parenthetical em-dashes (`---`) at one per sub-subsection. The recurring `X --- a Y` construction is a strong AI-voice tic that reads as decoration rather than as syntactic necessity. Replace with commas (apposition), colons (orientation), parentheses (true aside), or a sentence break.
**Why:** §3.5 round 1 had 7 parenthetical em-dashes across 8 sub-subsections — visible as a stylistic tic to a sensor reading for naturalness. The pattern is likely to repeat in any section using bulleted bullets-with-elaborations.
**When to apply:** Any section using bulleted iteration / finding / decision structure: §3.5, §4.5 multi-engine benchmark, §4.7 process documentation, §5.1.x discussion sub-sections, §5.4 L# limitations.
