## Deliverables (Inspera)

### 1. Thesis Report (Oppgave 1)
- **Format:** Single PDF document
- **Filename:** Three-digit task number, e.g., `001.pdf`
- **Must have:** Clickable table of contents with page numbers
- **Title:** Must be approved by supervisor. English title required. Title goes on diploma.
- **Abstract:** Short project description — entered in Inspera metadata
- **Norwegian abstract required:** Since thesis is in English, a Norwegian summary must be included (studieforskriften)

### 2. Attachments (Oppgave 2)
- **Format:** Single ZIP file (max 35 GB)
- **Contents:**
  - Presentation slides
  - Project handbook (prosjekthåndbok) — obligatorisk
  - **Visjonsdokument** — required for system development projects, own template
  - **Kravdokument** — functional requirements, own template
  - **Systemdokumentasjon** — technical system description, own template
  - Source code (or link to repository, agreed with supervisor)
  - **Poster** (English) — obligatorisk arbeidskrav
- **Source code:** Can be delivered as a link to a repository (e.g., GitHub) by agreement with supervisor

### 3. Individual Reflection Note (separate Inspera submission)
- **Required:** Yes — each group member submits their own
- **Content:** Individual assessment of own effort and learning
- **Filename:** Task number + student name, e.g., `001_MikaelStray.pdf`
- **Purpose:** Used by sensors to assess individual contributions
- **Important for grade differentiation** between group members

### 4. AI Declaration Form (deklareringsskjema)
- **Required:** Yes — must be attached to submission
- **Consequence if missing:** Grading will not begin
- **Content:** Description of how AI tools were used
- See section below for details

## Documentation Requirements

The following must be included in or with the thesis:

| Document | Required | Where |
|----------|:--------:|-------|
| Meeting minutes (all supervisor/client meetings) | Yes | Appendix or project handbook |
| Time sheets (timelister) | Yes | Project handbook |
| Status reports | Yes | Project handbook |
| Progress plan (Gantt chart) | Yes | Project handbook or Ch 3 |
| Clear attribution of who did what | Yes | Throughout thesis + reflection note |
| AI declaration form | Yes | Separate Inspera attachment |
| Presentation slides | Yes | Inspera attachment |

### Meeting minutes requirement
- Minutes from **all** project meetings are required
- The startup meeting minutes are especially important (agreements for the whole project)
- Minutes must be approved by supervisor
- Must document: who attended, task assignments, deadlines, follow-up from previous meeting

### Who-did-what requirement
- It must be **clearly apparent** who in the group did what
- This is documented through:
  - Individual reflection notes
  - Meeting minutes with task assignments
  - Time sheets per person
  - Attribution in the thesis text itself
- This is **especially important** for potential grade appeals
